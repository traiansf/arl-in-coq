Preliminaries

Reachability Logic

  • Program verification logic for K (Rosu, Ellison, and Schulte 2010)
  • Derives statements of the form φ ⇒ ψ
  • Two flavors: one-path and all-paths

One-path Reachability Logic (Rosu et al. 2013)

Whenever starting in a configuration matching φ there is always an execution path leading to a configuration matching ψ

All-paths Reachability Logic (Stefanescu et al. 2019)

A configuration matching ψ will be reached on any execution path starting in a configuration matching φ.

Reachability Logic and CTL

Resemblance

  • one-path reachability formula: φ ⟹ EF ψ
  • all-paths reachability formula: φ ⟹ AF ψ

Difference

  • CTL is richer?
  • Reachability formulae have structure and free variables

Reachability Logic and Matching Logic

Expressivity problem

  • Reachability formulae have structure and constraints

Solution: Enter Matching Logic

  • A logic mixing patterns and constraints
  • Specifically designed to express matching for K
  • Fixed-point operators (μ, ν) introduced to capture reachability
  • Formulae interpreted as sets
